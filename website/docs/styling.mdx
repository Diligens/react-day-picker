The package includes a CSS file to give a minimal style to DayPicker:

```jsx
import 'react-day-picker/lib/style.css';
```

There are two ways to customize the appearance:

1. using CSS classes;
2. using inline styles.

To each HTML element are assigned predefined `.rdp-*` CSS classes, as in the table below:

---

##### Elements Table

| Element name       | Default class          | Element description                          |
| ------------------ | ---------------------- | -------------------------------------------- |
| **container**      | `.rdp`                 | The container element                        |
| **caption**        | `.rdp-caption`         | The caption element                          |
| **day**            | `.rdp-day`             | The day element (a button when interactive)  |
|                    | `.rdp-day_selected`    | The day element when selected                |
|                    | `.rdp-day_disabled`    | The day element when disabled                |
|                    | `.rdp-day_today`       | The today element                            |
| **dayWrapper**     | `.rdp-day_wrapper`     | A wrapper of the day text element            |
| **month**          | `.rdp-month`           | Wrapper element of the month table           |
| **monthTable**     | `.rdp-month_table`     | The month's table                            |
| **monthTbody**     | `.rdp-month_tbody`     | The `tbody` of the monht's table             |
| **months**         | `.rdp-months`          | The container of the months                  |
| **head**           | `.rdp-head`            | The head element                             |
| **headRow**        | `.rdp-head_row`        | The head row element                         |
| **headWeekNumber** | `.rdp-head_weeknumber` | The head cell element for the week numbers   |
| **headWeekName**   | `.rdp-head_weekname`   | The head cell element for the weekday names  |
| **nav**            | `.rdp-nav`             | The navigation container element             |
| **navStart**       | `.rdp-nav_start`       | The button to navigate to the start day      |
| **navPrev**        | `.rdp-nav_prev`        | The button to navigate to the previous month |
| **navNext**        | `.rdp-nav_next`        | The button to navigate to the next button    |
| **week**           | `.rdp-week`            | The week row number                          |
| **weekNumber**     | `.rdp-weeknumber`      | The week's number cell                       |
| **weekDay**        | `.rdp-week_day`        | The week's day cell                          |

---

### Style Using CSS classes

To change the appearance using CSS classes, style `.rdp-*` classes in the application's CSS file.

```css
/* container */
.rdp {
  background: salmon:
}
/* head element */
.rdp-head {
  font-weight: bold;
  color: blue;
}
/* etc. */
```

### Defining Custom Class Names

Set custom class names

- if you use [CSS Modules](https://github.com/css-modules/css-modules) or other bundler that define their own class names;
- if you don't want to use the default class names in the table above.

To define custom class names, pass to the [`classNames`](./props#classnames) props the class name for each element.

```jsx preview
function App() {
  return (
    <>
      <style>{`.my-custom-head-class { color: salmon }`}</style>
      <DayPicker
        classNames={{
          container: 'my-custom-container-class',
          head: 'my-custom-head-class',
        }}
      />
    </>
  );
}
```

## Using Inline Styles

Use the [`styles`](./props#styles) prop to change the inline style of each element:

```jsx preview
function App() {
  return (
    <DayPicker
      showWeekNumber
      styles={{
        head: { color: 'salmon ' },
        weekNumber: { color: 'violet' },
      }}
    />
  );
}
```
